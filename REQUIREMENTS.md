# REQUIREMENTS.md - Picture Merge App v2 要件定義書（革新的アプローチ）

## 1. プロジェクト概要

### 1.1 アプリ名
**Picture Merge App v2** - AI対応 高品質写真合成Webアプリケーション

### 1.2 目的
2枚の人物写真から人物を高精度に切り抜き、同じ場所に一緒にいるかのように自然に合成する。
動画生成AI（Kling, Runway, Pika等）に渡す元画像として、「2人が一緒に写っている1枚の写真」として認識される品質を実現する。

### 1.3 背景
- 動画生成AIに結合画像を渡した際、「2人が一緒にいる」と認識される必要がある
- 単純な横並び結合ではAIが別々の写真として処理し、期待する動画が生成できない
- 遠近感・背景・照明の統一によるリアルな合成が必須

### 1.4 革新的アプローチのビジョン
バックエンド（Python）で全画像処理を高品質に実行し、フロントエンドはインタラクティブなUIに専念する。WebSocket接続によるリアルタイムプレビューで、パラメータ変更が即座に反映される体験を提供する。

## 2. 機能要件

### 優先度定義
| 優先度 | 意味 |
|--------|------|
| P0 | 必須（リリースブロッカー） |
| P1 | 重要（初回リリースに含める） |

### F1: 写真入力（2枚） — P0
**概要**: ドラッグ&ドロップまたはファイル選択で2枚の人物写真を入力

| 項目 | 仕様 |
|------|------|
| 入力方法 | ドラッグ&ドロップ / ファイル選択ダイアログ |
| 対応形式 | JPEG, PNG, WebP, HEIC/HEIF |
| 最大サイズ | 20MB/枚 |
| プレビュー | 入力後即座にサムネイル表示 |
| バリデーション | ファイル形式・サイズチェック、非画像ファイル拒否 |

**革新的要素**:
- HEIC/HEIFネイティブサポート（iPhone写真をそのまま使用可能）
- 画像メタデータ（EXIF）からの撮影条件自動読み取り（照明補正に活用）
- ドロップゾーンへのペースト（クリップボードから直接入力）対応

### F2: 人物セグメンテーション（自動） — P0
**概要**: バックエンドでrembg (U2Net)を使用し、高精度な人物切り抜きを自動実行

| 項目 | 仕様 |
|------|------|
| 処理エンジン | rembg (U2Net) on Python FastAPI |
| 出力 | 透過PNG（アルファマット付き） |
| レスポンス | 切り抜き画像 + バウンディングボックス + 足元Y座標 |
| 処理時間目標 | 1〜3秒/枚（M4チップ基準） |
| エッジ品質 | 髪の毛・半透明部分も自然に処理 |

**革新的要素**:
- アルファマット精密化: rembg出力のアルファチャンネルに対しガウシアンブラーで境界をなめらかに補正
- バウンディングボックス + 足元位置の自動検出をバックエンドで完結
- セグメンテーション結果のキャッシュ（同一画像の再処理回避）

### F3: 合成設定 — P1
**概要**: 合成パラメータをインタラクティブUIで調整可能

| パラメータ | 説明 | デフォルト | 範囲 |
|-----------|------|-----------|------|
| 背景色 | 合成時の背景色 | #FFFFFF（白） | カラーピッカー |
| 出力サイズ | 出力画像のサイズ | 1024x1024 | プリセット + カスタム |
| 人物1のX位置 | 左右の配置位置 | 30%（左寄り） | 0〜100% |
| 人物2のX位置 | 左右の配置位置 | 70%（右寄り） | 0〜100% |
| 人物1のスケール | 拡大縮小率 | 100%（自動算出） | 50〜200% |
| 人物2のスケール | 拡大縮小率 | 100%（自動算出） | 50〜200% |
| 人物のY位置 | 足元の高さ | 自動（足元揃え） | 手動上書き可 |
| 影の有無 | 足元影の表示 | ON | ON/OFF |
| 影の強度 | 影の濃さ | 50% | 0〜100% |
| 色調補正 | 2人の色温度統一 | ON | ON/OFF |

**革新的要素**:
- 出力サイズプリセット: 1024x1024（正方形）、1280x720（16:9）、720x1280（9:16）、カスタム
- 色調補正: OpenCVのヒストグラムマッチングで2人の写真の色温度・明るさを自動統一
- 影の強度・角度の細かい制御

### F4: 自動スケール調整 — P1
**概要**: 2人の身長が自然に見えるようスケールをインテリジェントに自動調整

| 項目 | 仕様 |
|------|------|
| 検出方法 | バウンディングボックス高さ比較 |
| 調整範囲 | 0.8〜1.2倍 |
| 基準 | 2人の足元を同一水平線に配置 |
| 上書き | 手動スケール設定で上書き可能 |

**革新的アルゴリズム**:
1. 両人物のバウンディングボックス高さを取得
2. 高さの比率を算出し、自然な範囲（0.8〜1.2）にクランプ
3. 頭頂-足元間の重心位置を考慮した配置
4. 横方向の間隔を「肩幅1つ分程度」に自動設定
5. 結果をフロントエンドに返却し、ユーザーが微調整可能

### F5: 合成プレビュー — P0
**概要**: WebSocket接続によるリアルタイム合成プレビュー

| 項目 | 仕様 |
|------|------|
| 更新方式 | WebSocket経由でリアルタイム更新 |
| レイテンシ | パラメータ変更から200ms以内にプレビュー反映 |
| ドラッグ操作 | Canvas上で人物をドラッグして位置調整 |
| プレビュー品質 | 低解像度でプレビュー → ダウンロード時にフル解像度 |

**革新的要素**:
- **WebSocket双方向通信**: パラメータ変更をWebSocket経由でバックエンドに送信し、合成結果をリアルタイムで返却
- **プレビュー用低解像度合成**: 512x512でプレビュー合成しレイテンシを最小化
- **Canvas直接操作**: React上のCanvas要素で人物の位置をドラッグ操作で微調整
- **デバウンス付きパラメータ送信**: スライダー操作中は150msデバウンスで過剰なリクエストを抑制

### F6: 出力・ダウンロード — P0
**概要**: 高品質な合成画像をPNG形式でダウンロード

| 項目 | 仕様 |
|------|------|
| 出力形式 | PNG（透過なし、背景色付き） |
| ファイル名 | `merged_{timestamp}.png` |
| 出力サイズ | 設定値に従う（デフォルト1024x1024） |
| 品質 | フル解像度で最終合成（プレビューとは別処理） |

**革新的要素**:
- バックエンドでPillow + OpenCVによるフル解像度合成
- 影生成: ガウシアンブラー楕円で自然な足元影
- 色調補正適用済みの最終出力

### F7: バッチ処理 — P2（Phase 2）
**概要**: 固定人物1枚 + 複数の人物写真で一括合成

| 項目 | 仕様 |
|------|------|
| 入力 | 固定人物1枚 + 複数枚（フォルダドロップ対応） |
| 処理 | バックグラウンドで順次合成 |
| 進捗表示 | プログレスバー + 個別サムネイル |
| 出力 | ZIP一括ダウンロード |

## 5. 合成アルゴリズム詳細（革新的アプローチ）

### 5.1 処理パイプライン

```
入力画像1, 入力画像2
    ↓
[Step 1] rembg セグメンテーション（並列処理）
    ↓
透過PNG1, 透過PNG2 + BBox + 足元座標
    ↓
[Step 2] 人物正規化
    - バウンディングボックスでクロップ
    - 足元座標を基準に正規化
    ↓
[Step 3] 色調補正（OpenCV）
    - ヒストグラムマッチング
    - 明度・彩度の統一
    ↓
[Step 4] スケール・位置計算
    - 高さ比率算出 → 0.8〜1.2倍にクランプ
    - 足元揃え + 横方向配置
    ↓
[Step 5] 影生成（OpenCV）
    - 足元位置にガウシアンブラー楕円
    - 影の強度・サイズを人物サイズに比例
    ↓
[Step 6] 最終合成（Pillow）
    - 背景キャンバス生成
    - 影レイヤー配置
    - 人物1, 人物2をアルファブレンディング
    - 出力サイズにリサイズ
    ↓
合成画像（PNG）
```

### 5.2 色調補正アルゴリズム

```python
# 概念コード: OpenCVヒストグラムマッチング
# 人物1の色調を基準に、人物2を補正
def match_color_histogram(source, reference):
    """LAB色空間でのヒストグラムマッチング"""
    src_lab = cv2.cvtColor(source, cv2.COLOR_BGR2LAB)
    ref_lab = cv2.cvtColor(reference, cv2.COLOR_BGR2LAB)
    # 各チャンネルの平均・標準偏差でマッチング
    for i in range(3):
        src_mean, src_std = src_lab[:,:,i].mean(), src_lab[:,:,i].std()
        ref_mean, ref_std = ref_lab[:,:,i].mean(), ref_lab[:,:,i].std()
        src_lab[:,:,i] = (src_lab[:,:,i] - src_mean) * (ref_std / src_std) + ref_mean
    return cv2.cvtColor(src_lab, cv2.COLOR_LAB2BGR)
```

### 5.3 影生成アルゴリズム

```python
# 概念コード: ガウシアンブラー楕円による影
def generate_shadow(canvas, foot_x, foot_y, person_width):
    """足元に自然な影を生成"""
    shadow_layer = np.zeros_like(canvas, dtype=np.uint8)
    shadow_width = int(person_width * 0.8)
    shadow_height = int(person_width * 0.15)
    cv2.ellipse(shadow_layer, (foot_x, foot_y),
                (shadow_width // 2, shadow_height // 2),
                0, 0, 360, (0, 0, 0), -1)
    shadow_layer = cv2.GaussianBlur(shadow_layer, (21, 21), 10)
    return cv2.addWeighted(canvas, 1.0, shadow_layer, 0.3, 0)
```

## 6. API設計

### 6.1 REST API

#### POST /api/segment
人物セグメンテーション（背景除去）

**Request**: `multipart/form-data`
- `image`: 画像ファイル（JPEG/PNG/WebP/HEIC）

**Response**: `application/json`
```json
{
  "segmented_image": "data:image/png;base64,...",
  "bbox": { "x": 50, "y": 10, "width": 400, "height": 900 },
  "foot_y": 900,
  "original_size": { "width": 1200, "height": 1600 },
  "processing_time_ms": 2100
}
```

#### POST /api/merge
フル解像度合成実行

**Request**: `application/json`
```json
{
  "image1": "data:image/png;base64,...",
  "image2": "data:image/png;base64,...",
  "settings": {
    "background_color": "#FFFFFF",
    "output_width": 1024,
    "output_height": 1024,
    "person1": { "x": 0.3, "y_offset": 0, "scale": 1.0 },
    "person2": { "x": 0.7, "y_offset": 0, "scale": 1.0 },
    "shadow": { "enabled": true, "intensity": 0.5 },
    "color_correction": true
  }
}
```

**Response**: `application/json`
```json
{
  "merged_image": "data:image/png;base64,...",
  "processing_time_ms": 450
}
```

#### GET /api/health
ヘルスチェック

**Response**:
```json
{
  "status": "ok",
  "rembg_loaded": true,
  "version": "2.0.0"
}
```

## 8. リスクと対策

### 8.1 技術リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| rembgのセグメンテーション精度不足 | 高 | 低 | U2Netは実績あり。アルファマット後処理で補正。Apple Vision Frameworkをフォールバックに準備 |
| HEIC形式のデコード問題 | 中 | 中 | pillow-heifライブラリで対応。非対応時はJPEG/PNG変換を促す |
| WebSocket接続の安定性 | 中 | 低 | 切断時の自動再接続。REST APIフォールバック |
| M4チップでの処理速度 | 低 | 低 | Neural Engine非活用でも3秒以内達成可能。必要時Apple Vision切り替え |
| メモリ使用量超過 | 中 | 低 | 画像リサイズ戦略（入力画像の最大解像度制限: 長辺4000px） |

### 8.2 UXリスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| 合成結果が不自然 | 高 | 中 | 色調補正・影生成・スケール自動調整で最小化 |
| 操作の複雑さ | 中 | 低 | デフォルト設定で十分な品質。詳細設定は折りたたみ |
| 処理待ち時間のストレス | 中 | 低 | プログレス表示 + 段階的プレビュー |

### 8.3 プロジェクトリスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| OpenCV依存のビルド問題 | 中 | 中 | opencv-python-headlessを使用（GUI不要） |
| Python環境の差異 | 低 | 低 | venvによる仮想環境 + requirements.txt固定 |

| ブラウザ | Chrome / Edge（最新版） |
| デバイス | デスクトップ専用（モバイル対応不要） |

## 11. 革新的アプローチまとめ

### 従来アプローチとの差別化

| 項目 | 従来（v1） | 革新的（v2） |
|------|-----------|-------------|
| 合成処理 | フロントエンドCanvas | バックエンドPillow + OpenCV |
| 色調補正 | なし | OpenCVヒストグラムマッチング |
| 影生成 | CSSシャドウ | ガウシアンブラー楕円（OpenCV） |
| プレビュー | 手動更新 | WebSocketリアルタイム |
| 位置調整 | スライダーのみ | Canvas上ドラッグ操作 |
| スケール | 手動設定 | インテリジェント自動調整 |
| セグメンテーション | フロントエンド処理 | バックエンドrembg（高精度） |
| HEIC対応 | なし | ネイティブサポート |

### 技術的優位性
1. **バックエンド集約型処理**: Python画像処理ライブラリのフルパワーを活用
2. **WebSocketリアルタイムプレビュー**: パラメータ調整が即座に反映
3. **OpenCV色調補正**: LAB色空間でのヒストグラムマッチングにより2枚の写真の雰囲気を統一
4. **高品質影生成**: ガウシアンブラー楕円で自然な足元影を実現
5. **インタラクティブUI**: Canvas上でのドラッグ操作による直感的な位置調整
